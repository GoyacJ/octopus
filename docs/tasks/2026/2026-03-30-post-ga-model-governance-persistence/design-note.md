# Design Note

- Problem:
  - The repository already freezes the first five model-governance contracts, but no implemented slice yet persists provider/catalog/profile/policy truth or records per-run `ModelSelectionDecision` state.
- Goal:
  - Deliver the smallest approved consumer implementation that persists model-governance truth in `crates/governance` and run-scoped `ModelSelectionDecision` records in `crates/runtime`, without introducing transport or UI changes.
- Acceptance Criteria:
  - `crates/governance` owns the durable storage path for `ModelProvider`, `ModelCatalogItem`, `ModelProfile`, and `TenantModelPolicy`.
  - `crates/runtime` owns run-scoped recording and retrieval of `ModelSelectionDecision`.
  - Owner docs reflect the new implemented slice while keeping transport/UI follow-ons deferred.
- Non-functional Constraints:
  - Keep the existing five model-governance contracts unchanged.
  - Preserve the current shared SQLite database and runtime migration pattern.
  - Avoid new crate boundaries or new external dependencies.
- MVP Boundary:
  - Add the minimum migration and record structs needed for persistence.
  - Add targeted upsert/fetch/list behavior in `crates/governance`.
  - Add targeted record/fetch behavior in `crates/runtime`.
  - Do not add remote read surfaces, client methods, or app-local model state.
- Layer Placement:
  - Shared contract truth remains in `schemas/governance`.
  - Durable provider/catalog/profile/policy persistence lives in `crates/governance`.
  - Run-scoped decision storage and readback live in `crates/runtime`.
  - Owner docs and task packaging live under `docs/`.
- Module Boundaries:
  - `crates/governance` defines and persists `ModelProviderRecord`, `ModelCatalogItemRecord`, `ModelProfileRecord`, and `TenantModelPolicyRecord`.
  - `crates/runtime` defines and persists `ModelSelectionDecisionRecord`, wires it into run-scoped read paths, and keeps the decision record separate from catalog truth.
  - `apps/remote-hub`, `packages/hub-client`, and `apps/desktop` remain unchanged in this slice.
- Inputs:
  - Existing model-governance schemas under `schemas/governance`.
  - The design-only `2026-03-30-post-ga-model-governance-consumers` package.
  - The existing runtime SQLite migration stack and governance store patterns.
- Outputs:
  - One runtime migration extending the shared database.
  - Governance persistence methods and tests.
  - Runtime decision-record methods and tests.
  - Owner-doc truth updates.
- State Transitions:
  - No new public runtime state machine is introduced.
  - `ModelSelectionDecision` is persisted as a bounded per-run record and does not alter `Run` state transitions.
- Error Handling:
  - Governance persistence methods return store errors on SQL failures or missing lookups.
  - Runtime decision recording fails if the target run does not exist.
  - If a decision already exists for a run, the runtime returns the persisted record instead of silently creating duplicates.
- Tech Stack Decision:
  - Reuse the existing Rust, SQLx, SQLite, and task-package stack. No new tech is introduced.
- Visual Framework Impact:
  - None.
- Human Approval Points:
  - Stop and return to task-package design if implementation reveals a need for new transport DTOs or provider/capability runtime redesign.
- Reused Components:
  - `SqliteGovernanceStore`
  - `Slice2Runtime` / `RunOrchestrator`
  - Existing runtime migration pattern
  - Existing schema-ts contract verification path
- New Abstractions:
  - Record structs for the four governance persistence objects and one runtime decision record.
  - No new service layer beyond the existing governance store and runtime orchestrator.
- Trade-offs:
  - Deferring transport and UI consumers keeps this slice small and preserves the boundary frozen by ADR 0007 and the design-only predecessor package.
- Test Strategy:
  - Add governance-store integration tests that run against the shared runtime migration stack.
  - Add runtime integration tests that prove per-run decision recording is persisted and reloaded correctly.
  - Keep schema-ts validation green to prove the existing contracts remain compatible.
- ADR Needed:
  - No. ADR 0007 and the design-only predecessor package already freeze the durable boundary.
