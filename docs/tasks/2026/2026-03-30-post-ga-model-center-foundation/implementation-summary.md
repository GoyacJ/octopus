- Structure Decision:
  - Keep the first Model Center truth narrowly inside `schemas/governance`, `packages/schema-ts`, ADR 0007, and owner-doc/task-package updates instead of distributing parallel model definitions across runtime or surface code.
- Why This Structure:
  - The repository needed one canonical vocabulary and one minimal contract set before any persistence, transport, or provider-adapter consumer work could proceed safely.
- Reused Patterns:
  - Reused the existing JSON Schema source-of-truth boundary, `packages/schema-ts` AJV registry pattern, ADR indexing flow, and the docs/reference separation already used elsewhere in the repository.
- New Dependencies:
  - None. This slice stays within the existing documentation plus `schema-ts` validation stack.
- Error Handling Strategy:
  - Keep overlapping supplement terms mapped into PRD vocabulary, reject runtime/provider expansion in this slice, and rely on `schema-ts` parser tests plus owner-doc alignment to catch drift.
- Deferred Items:
  - Rust-side persistence and evaluation for model governance.
  - Runtime consumption of `ModelSelectionDecision`.
  - Read-only transport surfaces, provider connectivity, provider built-in tool modeling, and any `ProviderAdapter` SPI.
- Non-goals Preserved:
  - No Rust runtime behavior change.
  - No remote-hub endpoint or desktop/web surface change.
  - No `CapabilityResolver`, `ToolSearch`, or `SkillPack` refactor.
