## Contract Change

- Change Type:
  - Shared Schema
  - Public API
  - Internal Interface
  - Cross-language Contract
- New / Updated Schemas:
  - Update `schemas/governance/approval-request.schema.json` so `approval_type` supports `execution | knowledge_promotion` and `target_ref` is required.
  - Update `schemas/observe/inbox-item.schema.json` and `schemas/observe/notification.schema.json` so `target_ref` is required.
  - Add `schemas/observe/request-knowledge-promotion-command.schema.json`.
- New / Updated Commands:
  - `RequestKnowledgePromotionCommand`
  - Existing `ResolveApprovalCommand` now resolves both execution and knowledge-promotion approvals through one surface.
- New / Updated Queries:
  - Approval detail query by `approval_id`.
  - Knowledge-promotion request by `candidate_id`.
- New / Updated Events:
  - None; the slice reuses the existing hub connection and surface refresh behavior.
- New / Updated DTOs:
  - `ApprovalRequest`
  - `InboxItem`
  - `Notification`
- Compatibility Impact:
  - Existing execution approval behavior remains compatible, but all approval-centric consumers must now accept `target_ref` and the expanded `approval_type`.
  - Existing inbox / notification consumers must treat approval items as target-aware rather than execution-only.
  - Shared client and remote routes gain approval detail and request-promotion operations without removing the existing approval resolve operation.
- Affected Consumers:
  - `crates/governance`
  - `crates/observe-artifact`
  - `crates/runtime`
  - `packages/schema-ts`
  - `packages/hub-client`
  - `apps/remote-hub`
  - `apps/desktop`
- Migration Notes:
  - Add a runtime SQLite migration that backfills `approval_requests.target_ref` to `run:<run_id>` for existing execution approvals.
  - Backfill `inbox_items.target_ref` and `notifications.target_ref` from the owning approval when possible, falling back to `run:<run_id>`.
  - Add an index on `(approval_type, target_ref, status)` so one candidate can reuse its open pending promotion approval.
- Generation Impact:
  - `packages/schema-ts` must register the new command schema and expose the updated approval / inbox / notification parsers and types.
- Open Questions:
  - Whether a future standalone governance center should consume the same approval detail contract directly or introduce a richer read model in a later slice.
