## Implementation Summary

- Goal:
  - Implement the first GA minimum surface foundation on top of the verified local runtime.
- Files Added:
  - Shared Hub surface schemas for task create, run summary/detail, approval resolve, inbox/notification, artifact summary, knowledge summary/detail/promote, hub events, hub connection status, and project context.
  - `packages/schema-ts` as the schema-derived TypeScript validation/contract-consumption package.
  - `packages/hub-client` as the transport-neutral client boundary with local `invoke + event` and remote `HTTP + SSE` adapters.
  - `apps/remote-hub` as the thin `axum` assembly shell over the runtime.
  - `apps/desktop` as the minimum Vue 3 + Pinia shell with workspace/task-create and run-detail surfaces.
  - ADR 0006 and the task-package artifacts for this phase.
- Files Changed:
  - `Cargo.toml` and `Cargo.lock` to register the remote-hub app and supporting workspace dependencies.
  - `package.json`, `pnpm-lock.yaml`, and `tsconfig.base.json` to register workspace-level TypeScript verification and shared TS configuration.
  - `crates/runtime/src/lib.rs` and `crates/runtime/src/services.rs` to expose thin runtime wrapper reads required by the remote-hub shell.
  - `crates/runtime/tests/schema_contracts.rs`, `schemas/observe/audit-record.schema.json`, and `schemas/observe/trace-record.schema.json` to align shared schema truth with already-emitted knowledge/audit runtime events.
  - `docs/architecture/VISUAL_FRAMEWORK.md`, `docs/tasks/README.md`, and `docs/decisions/README.md` to freeze the minimum GA surface and link the new delivery artifacts.
- Files Removed:
  - None.
- Structure Decision:
  - Keep shared DTOs in `schemas/`, shared TS consumption in `packages/`, and surface assembly in `apps/`.
- Why This Structure:
  - It preserves the existing monorepo ownership rules and prevents app-local logic from becoming shared truth.
- Reused Patterns:
  - Task-package documentation flow, schema-first validation, runtime API reuse, and thin-assembly boundaries for both Rust and TypeScript app layers.
- New Dependencies:
  - `packages/schema-ts`: `ajv`, `ajv-formats`.
  - `apps/remote-hub`: workspace `axum`, `futures-util`, `http-body-util`, and `tower`.
  - `apps/desktop`: `vue`, `vue-router`, `pinia`, `vite`, `vitest`, and Vue test/build tooling declared in the app package.
- Error Handling Strategy:
  - Normalize local/remote client failures into one `HubClientTransportError` shape in `packages/hub-client`.
  - Preserve runtime failures as structured HTTP errors in `apps/remote-hub`.
  - Surface workspace/run loading and task-submit failures through the desktop store instead of assuming success in UI routes.
- Deferred Items:
  - Actual Tauri runtime bridge registration, remote auth/deployment, richer page decomposition, trigger expansion, and real MCP transport remain deferred.
- Non-goals Preserved:
  - No Beta collaboration, no vector retrieval, no connector-management UI, no Org Graph work, and no parallel runtime logic in apps.
