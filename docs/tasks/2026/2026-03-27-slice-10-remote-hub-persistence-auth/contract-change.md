## Contract Change

- Change Type:
  - Shared Schema
  - Internal Interface
  - Persistence Model
- New / Updated Schemas:
  - Add remote login command, remote session summary, and remote auth error DTO under `schemas/interop`.
  - Extend `interop/hub-connection-status.schema.json` with auth-aware state fields.
- New / Updated Commands:
  - Remote login command.
  - Remote logout command if needed for session invalidation.
- New / Updated Queries:
  - Remote current-session query.
- New / Updated Events:
  - None required in MVP; existing `hub.connection.updated` payload shape is extended.
- New / Updated DTOs:
  - Hub connection status gains auth state.
  - Hub client transport error gains discriminated auth failure kind.
- Compatibility Impact:
  - Existing local-mode client flows remain compatible.
  - Remote consumers must tolerate the new auth-aware fields and login/session surface.
- Affected Consumers:
  - `packages/schema-ts`
  - `packages/hub-client`
  - `apps/desktop`
  - `apps/remote-hub`
- Migration Notes:
  - Add auth/access tables to remote-hub SQLite and seed a bootstrap user plus membership record.
- Generation Impact:
  - `packages/schema-ts` needs to expose validators/types for the new interop contracts.
- Open Questions:
  - Whether session revocation semantics need a durable ADR after implementation.
