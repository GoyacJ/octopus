## Contract Change

- Change Summary:
  - Add one shared read method to the cross-language desktop/hub surface: `HubClient.listRuns(workspaceId, projectId): Promise<RunSummary[]>`.
  - Add one interop-owned local transport command: `list_runs`.
  - Add one remote-hub project-scoped recent-runs read route.
- Why:
  - Slice 14 needs a dedicated `Runs` surface, but the tracked codebase currently has no shared project-scoped recent-run list API even though `RunSummary` already exists.
- Shared Contract Impact:
  - Reuse the existing `RunSummary` JSON schema and TypeScript parser.
  - Extend `LocalHubTransportContract.commands` with `list_runs`.
  - Extend the shared `HubClient` interface with `listRuns(workspaceId, projectId)`.
- Runtime / Transport Impact:
  - Local transport must map `hub:list_runs` to a runtime query scoped by `workspace_id + project_id`.
  - Remote-hub must expose the same recent-run read semantics over HTTP.
  - Both transports must preserve the same ordering and empty-state behavior.
- Query Semantics:
  - Scope by `workspace_id` and `project_id`.
  - Order by most recently updated first, then deterministic tie-breaker.
  - Return only a small recent window.
  - No pagination, filtering, board aggregation, or alternate DTO shapes in this slice.
- Compatibility:
  - Existing `RunSummary` payload shape remains unchanged.
  - Existing `HubEvent` shape remains unchanged.
  - Existing task start, run detail, approval, automation, notification, and connection APIs remain unchanged.
- Explicit Deferrals:
  - `retryRun`, `terminateRun`, or other run control writes.
  - Streaming/chat output or new `InteractionPrompt` / `DiscussionSession` contracts.
  - New inbox/notification event contracts, pagination contracts, or board/list aggregation DTOs.
