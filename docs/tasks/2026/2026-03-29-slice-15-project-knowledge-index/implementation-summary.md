- Structure Decision:
  - Keep one additive shared DTO (`ProjectKnowledgeIndex`) and one additive local command / remote route, with runtime owning the mixed candidate/asset aggregation and desktop owning only route/store/view composition.
- Why This Structure:
  - It preserves `schemas/` as the only cross-language truth, avoids introducing a second knowledge row model, and keeps transport layers thin instead of duplicating aggregation or scoping semantics.
- Reused Patterns:
  - Reused existing `KnowledgeSummary` entry shape, existing project-scoped local/remote read parity patterns, and the Slice 14 desktop route-split store/view structure.
- New Dependencies:
  - None.
- Error Handling Strategy:
  - Missing project knowledge space stays a runtime/transport not-found path.
  - Remote-hub continues to enforce auth and workspace membership before delegating to runtime.
  - Desktop knowledge view stays read-only and surfaces traceability via existing source refs rather than inventing new governance state.
- Deferred Items:
  - Workspace-wide knowledge boards.
  - Vector retrieval and Org Graph promotion.
  - `retry_run` / `terminate_run`.
  - Tenant / RBAC / IdP expansion.
- Non-goals Preserved:
  - No new knowledge-page governance action entry point.
  - No new event contract.
  - No app-local DTO truth in desktop or transport layers.
