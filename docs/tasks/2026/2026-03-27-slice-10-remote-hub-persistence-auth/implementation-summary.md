- Structure Decision:
  - Put remote identity, membership, password hashing, session persistence, and JWT logic in a dedicated Rust crate and keep `apps/remote-hub` as HTTP/SSE assembly only.
- Why This Structure:
  - The slice needed a durable access boundary without creating a second execution shell or moving runtime truth into the app layer.
- Reused Patterns:
  - Reused the existing SQLite-backed slice pattern, schema-first contract flow, and remote-hub integration-test style already established in earlier slices.
- New Dependencies:
  - Rust workspace: `argon2`, `jsonwebtoken`
- Error Handling Strategy:
  - Normalize missing or invalid bearer tokens to `auth_required`, expired tokens to `token_expired`, and workspace mismatch to `workspace_forbidden`, while keeping transport disconnect in `HubConnectionStatus.state`.
- Deferred Items:
  - Full tenant/RBAC management.
  - Automation-management surface.
  - External IdP integration.
- Non-goals Preserved:
  - No second execution shell.
  - No automation UI expansion.
