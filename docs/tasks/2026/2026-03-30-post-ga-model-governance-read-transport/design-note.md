# Design Note

- Problem:
  - The repository now persists model-governance truth and run-scoped `ModelSelectionDecision` records in Rust, but no read-only transport slice has yet frozen how those records should be surfaced to remote-hub, shared clients, or desktop consumers.
- Goal:
  - Define the smallest next slice that exposes model-governance records through read-only transport paths without introducing writes, provider connectivity, or UI editing behavior.
- Acceptance Criteria:
  - `apps/remote-hub` is limited to read-only route assembly for model-governance queries.
  - `packages/hub-client` exposes transport-neutral read accessors only.
  - `apps/desktop` is limited to read-only consumption of provider/catalog/profile/policy records and run-scoped `ModelSelectionDecision`.
  - No write routes, no write DTOs, and no provider connectivity work are authorized by this package.
- In Scope:
  - Read-only transport/module ownership.
  - Read-only query shape decisions.
  - Promotion gates for a later implementation slice.
- Out Of Scope:
  - Write behavior, governance editing, tenant administration, provider credentials, provider connectivity, built-in tool modeling, or runtime redesign.
- Module Boundaries:
  - `crates/governance` remains the truth owner for provider/catalog/profile/policy persistence.
  - `crates/runtime` remains the truth owner for run-scoped `ModelSelectionDecision` recording and readback.
  - `apps/remote-hub` only assembles read-only routes and does not create new business truth.
  - `packages/hub-client` only wraps read-only transport access and stays transport-neutral.
  - `apps/desktop` only consumes the read-only transport shape and does not own model-governance truth.
- Inputs:
  - The five existing model-governance contracts under `schemas/governance`.
  - The design-only `2026-03-30-post-ga-model-governance-consumers` package.
  - The implemented `2026-03-30-post-ga-model-governance-persistence` package.
- Outputs:
  - One bounded design-only task package for the read-only transport follow-on.
- State Transitions:
  - No new runtime or governance state machine changes are introduced.
- Error Handling:
  - Any need for summary/detail DTOs or transport-only projections is a hard stop that must be handled by a bounded contract-change before implementation.
- Tech Stack Decision:
  - Reuse the existing remote-hub, hub-client, and desktop stacks. No new technology is introduced.
- Human Approval Points:
  - Stop and return to task-package design if any proposed read path requires write semantics, provider connectivity, or new cross-language contracts.
- Reused Components:
  - Existing runtime/governance persistence boundary.
  - Existing remote-hub route assembly and `packages/hub-client` read-pattern conventions.
- New Abstractions:
  - None yet. This package is design-only.
