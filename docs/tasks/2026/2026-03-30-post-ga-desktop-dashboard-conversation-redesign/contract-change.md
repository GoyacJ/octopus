# Contract Change

- Change Type:
  - App-local Interface
  - Route Topology
  - Owner-Document Update
- New / Updated Schemas:
  - None. This slice adds no new cross-language shared schema truth.
- New / Updated Commands:
  - None
- New / Updated Queries:
  - None
- New / Updated Events:
  - None
- New / Updated DTOs:
  - None across the shared contract surface.
  - App-local desktop-only additions:
    - `PreferenceState { locale, themeMode }`
    - `ConversationStage = drafting | proposal_ready | execution_started | follow_up`
    - `ConversationDraft`
    - `ExecutionProposal`
    - `ConversationThreadSummary`
- Route Changes:
  - Add project-scoped `Dashboard`.
  - Add project-scoped `Conversation`.
  - Keep `Tasks` as an expert-only direct execution route.
  - Remove `Notifications` and `Models` from first-class primary navigation while preserving their authority access as secondary/reminder surfaces.
- Compatibility Impact:
  - No cross-language compatibility change.
  - Existing desktop entry semantics change locally from `Tasks-first` to `Dashboard-first` for active projects.
  - Conversation drafts and preferences persist only in local desktop storage and do not affect shared runtime compatibility.
- Affected Consumers:
  - `apps/desktop`
  - desktop tests under `apps/desktop/test`
  - owner docs and ADR indexes
- Migration Notes:
  - No database or transport migration is required.
  - Existing local desktop profiles remain valid; the remembered project now resolves to `Dashboard` instead of `Tasks`.
- Generation Impact:
  - None. `schemas/` and `packages/schema-ts` remain unchanged in this slice.
- Open Questions:
  - Cross-device conversation continuity remains intentionally deferred to a later schema-first slice if product pressure proves it necessary.
