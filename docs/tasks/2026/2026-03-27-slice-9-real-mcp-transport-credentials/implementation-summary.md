- Structure Decision:
  - Keep one `McpGateway` boundary and split execution internally by transport kind: `simulated` for existing fake-path tests and `http_jsonrpc` for Slice 9 real transport.
- Why This Structure:
  - It preserves the existing runtime orchestration shell and SQLite-backed audit path while localizing real transport, credential resolution, and health mutation inside `crates/interop-mcp`.
- Reused Patterns:
  - Existing environment-lease lifecycle, invocation persistence, runtime connector dispatch, artifact provenance, and trust-gate semantics.
- New Dependencies:
  - `reqwest` for outbound HTTP JSON-RPC calls.
- Error Handling Strategy:
  - Normalize credential, auth, timeout, connect, non-2xx, invalid JSON, and JSON-RPC error conditions inside `McpGateway` into persisted failed invocations with explicit retryability.
- Deferred Items:
  - Richer remote-hub auth/persistence.
  - Desktop or Hub connector-management surfaces.
  - Additional transports such as stdio or SSE.
- Non-goals Preserved:
  - No new Desktop page.
  - No `packages/hub-client` connector-management API.
  - No second execution shell outside `RunOrchestrator`.
