## Delivery Note

- What Changed:
  - Added the Slice 11 governance interaction contract set, runtime approval-driven knowledge-promotion path, remote-hub approval detail / request-promotion routes, hub-client parity methods, desktop inline governance actions, and owner-doc truth alignment.
- Why:
  - The governed runtime already persisted approvals and knowledge candidates, but GA still lacked one formal human-governed interaction path to inspect approvals and promote shared knowledge without bypassing governance.
- User / System Impact:
  - Workspace and Run surfaces can now load approval detail, resolve execution and knowledge-promotion approvals inline, and request knowledge promotion through the same shared client surface in local and remote modes.
- Risks:
  - Desktop local mode still depends on an external bridge registration that is not implemented in the tracked repository.
  - Direct `promoteKnowledge(...)` remains as an internal primitive path, so future cleanup may still need to narrow usage further around the approval-driven flow.
- Rollback Notes:
  - Revert the Slice 11 migration and the coupled schema / runtime / remote-hub / hub-client / desktop changes together; the approval target and promotion flow changes cross one shared contract boundary.
- Follow-ups:
  - Decide later whether a standalone governance center, richer notification handling, or deeper knowledge-governance workflows merit their own task packages.
