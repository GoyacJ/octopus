## Design Note

- Problem:
  - The tracked repository already proves the governed runtime, shared contracts, shared client, remote-hub surface, automation surface, and governance explainability, but the desktop local path still stops at a test seam on `window.__OCTOPUS_LOCAL_HUB__`.
- Goal:
  - Add a real Tauri 2 local host so the desktop shell can consume the existing runtime truth through the already-frozen `HubClient` boundary without expanding GA scope.
- Acceptance Criteria:
  - The host opens the local SQLite runtime, runs migrations through the existing runtime-open path, seeds deterministic demo context on first boot, and exposes the current desktop surface operations through Tauri commands.
  - TypeScript and Rust consume one interop-owned local command/event directory from `schemas/interop`.
  - Desktop bootstrap no longer depends on handwritten window injection in tracked runtime usage.
  - Local mode supports `manual_event` and cron ticking, while rejecting unsupported external-ingress trigger types.
- Non-functional Constraints:
  - No shared business logic moves into `apps/desktop`.
  - No app-local DTO truth is introduced.
  - No local auth/session model is added.
  - The desktop store remains primarily pull-driven.
- MVP Boundary:
  - One local runtime database path owned by the Tauri host.
  - One deterministic demo workspace/project seed.
  - One seeded capability/binding/grant/budget path reused for task and automation explainability.
  - Event support is limited to the already tracked `HubEvent` contract.
- Layer Placement:
  - Local transport owner constants live in `schemas/interop`.
  - TypeScript-side parsing/export of those constants lives in `packages/schema-ts`.
  - Shared client transport wiring lives in `packages/hub-client`.
  - Tauri frontend adapter wiring lives in `apps/desktop/src`.
  - The Tauri Rust host lives in `apps/desktop/src-tauri`.
- Module Boundaries:
  - `crates/runtime` remains the owner of task, automation, governance, approval, artifact, knowledge, and cron semantics.
  - `apps/desktop/src-tauri` may seed default data, open runtime state, map commands to runtime calls, and emit schema-owned hub events, but it may not redefine business rules.
  - `apps/desktop/src` may register a Tauri local transport and render local-mode restrictions, but it may not bypass `HubClient`.
- Inputs:
  - Existing `Slice2Runtime` APIs for project context, task/run flow, automation lifecycle/dispatch, governance, artifacts, knowledge, and cron tick.
  - Existing shared surface schemas already consumed by `packages/schema-ts` and `packages/hub-client`.
  - ADR 0006 transport parity rules.
- Outputs:
  - Shared local transport contract constants.
  - Real Tauri invoke/event bridge registration.
  - Tracked Rust host assembly and deterministic local seed path.
  - Desktop UI restrictions for unsupported local trigger kinds.
- State Transitions:
  - No new core runtime states are introduced.
  - Connection state in local mode is presentation-only and fixed to `mode=local`, `state=connected`, `auth_state=authenticated`.
  - Unsupported local trigger types fail fast at creation time instead of creating dormant automations.
- Error Handling:
  - Host command failures surface explicit error messages back through Tauri invoke rejection.
  - Unsupported `webhook` and `mcp_event` create attempts must produce clear local-mode error text.
  - Bootstrap failure remains explicit if no local transport bridge is registered.
- Reused Components:
  - Existing runtime migration/open path.
  - Existing runtime list/detail/approval/knowledge methods.
  - Existing `HubClient` local adapter shape.
  - Existing desktop store and routed surface composition.
- New Abstractions:
  - Shared local transport owner contract.
  - Tauri local transport registration adapter.
  - Desktop local host assembly with deterministic seed.
- Trade-offs:
  - The slice keeps response mapping thin inside the host instead of broadening the runtime with a second app-focused read model.
  - Unsupported local ingress is rejected now rather than partially stubbed, which keeps GA behavior honest.
  - The host emits only the current minimum event surface because the UI remains pull-first.
- Test Strategy:
  - Write failing Rust host tests first for seed, round-trip, approval resume, automation/manual dispatch, and cron tick.
  - Write failing TypeScript tests first for shared transport constants, Tauri bridge registration, and local-mode trigger restrictions in the desktop shell.
  - Re-run full workspace verification after implementation.
- ADR Needed:
  - Not by default. ADR 0006 already owns local-vs-remote transport parity. Add a new ADR only if the local transport owner contract introduces a durable new repository-wide rule beyond that decision.
