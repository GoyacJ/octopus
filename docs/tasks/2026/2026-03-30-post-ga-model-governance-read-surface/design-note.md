# Design Note

- Problem:
  - The repository now persists model-governance truth in Rust and records one run-scoped `ModelSelectionDecision` per run, but no verified transport or desktop surface yet exposes that truth to authenticated consumers.
- Goal:
  - Deliver the smallest read-only implementation slice that exposes model-governance records through shared contracts, local/remote transports, and desktop read surfaces without reopening ownership or widening into writes.
- Acceptance Criteria:
  - `RunDetail` exposes an optional `model_selection_decision`.
  - Provider/catalog/profile/policy reads remain separate read-only surfaces and do not introduce write commands.
  - Remote and local transports expose the same read-only methods through `HubClient`.
  - Desktop adds a workspace-level `Models` surface and keeps `RunView` authoritative for run detail.
- Non-functional Constraints:
  - Reuse the existing five model-governance contracts as the only cross-language truth objects.
  - Keep `apps/remote-hub` and `apps/desktop/src-tauri` as thin assembly layers.
  - Keep `apps/desktop/src` on the existing store and route-loader patterns.
- MVP Boundary:
  - Workspace-scoped provider/catalog/profile reads.
  - One workspace-scoped effective policy read.
  - One additive run-detail field for `ModelSelectionDecision`.
  - One read-only desktop `Models` view and one read-only `RunView` augmentation.
- Layer Placement:
  - `crates/governance` remains the truth owner for `ModelProvider`, `ModelCatalogItem`, `ModelProfile`, and `TenantModelPolicy`.
  - `crates/runtime` remains the truth owner for run-scoped `ModelSelectionDecision` recording and readback.
  - `packages/hub-client` owns transport-neutral read methods only.
  - `apps/remote-hub` and `apps/desktop/src-tauri` expose read-only assembly only.
  - `apps/desktop` consumes read-only state and must not create a second truth layer.
- Module Boundaries:
  - Provider/catalog/profile reads are list surfaces.
  - Policy is exposed as one workspace-scoped effective policy read, not as a tenant-admin enumeration surface.
  - `Models` is a workspace-level route alongside `Projects`, `Inbox`, `Notifications`, and `Connections`.
  - Run-scoped decision visibility stays anchored in `RunView`, with `Models` linking back to the current or relevant run context rather than becoming a second run-detail authority.
- Inputs:
  - ADR 0007.
  - `2026-03-30-post-ga-model-governance-read-transport`.
  - Existing model-governance schemas and persistence/runtime APIs.
- Outputs:
  - Additive shared contract changes for local transport and `RunDetail`.
  - Read-only local and remote transport parity.
  - Shared `HubClient` accessors for model-governance reads.
  - Desktop read-only `Models` and `RunView` consumption.
- Error Handling:
  - Degraded, offline, forbidden, and empty states must remain explicit and read-only in desktop copy.
  - If implementation proves a new summary/detail DTO is required, stop and add a bounded contract-change note instead of inventing new truth ad hoc.
- Tech Stack Decision:
  - No new stack. Stay within Rust, SQLite, JSON Schema, TypeScript, Vue 3, Pinia, and the existing Tauri/remote-hub transport layers.
- Visual Framework Impact:
  - One additional workspace-level read-only view in the existing desktop shell.
- Human Approval Points:
  - None.
- Reused Components:
  - Existing model-governance schemas.
  - Existing `RunDetail` and `HubClient` patterns.
  - Existing desktop workspace/project IA split.
- New Abstractions:
  - No new cross-language truth objects.
  - Only additive store/view state needed to consume the existing contracts.
- Trade-offs:
  - Keeping policy as a singular effective read avoids widening into tenant admin enumeration, but means richer admin visibility remains a later slice.
  - Reusing `RunDetail` avoids a second run-decision endpoint, but limits run-scoped model-decision visibility to existing run detail loading paths.
- Test Strategy:
  - Contract tests for `RunDetail` and local transport growth.
  - Shared `HubClient` parity tests for local and remote model-governance reads.
  - Remote-hub auth/membership tests for the new routes.
  - Desktop local-host parity tests.
  - Desktop view tests for `Models` and `RunView`.
- ADR Needed:
  - No. ADR 0007 already freezes the durable terminology and ownership boundary.
