## Design Note

- Problem:
  - GA promises external event-triggered automation, but the tracked repo has no governed webhook ingress.
- Goal:
  - Add one thin remote-hub webhook entrypoint that validates request identity and projects events into the shared delivery path.
- Acceptance Criteria:
  - Webhook trigger metadata includes ingress mode and secret hash.
  - Remote-hub rejects requests without a valid idempotency key and shared secret.
  - Accepted webhook requests create or reuse one trigger delivery and governed automation run.
- Non-functional Constraints:
  - No plaintext secret persistence.
  - No business branching in the HTTP handler beyond validation and runtime dispatch.
- MVP Boundary:
  - One POST route, opaque JSON body, fixed headers.
  - No public management surface.
- Layer Placement:
  - Webhook metadata contracts stay in `schemas/runtime`.
  - Secret validation and dispatch primitives stay in `crates/runtime`.
  - HTTP ingress stays in `apps/remote-hub`.
- Module Boundaries:
  - `AutomationIntake` owns trigger metadata persistence.
  - `Slice2Runtime` owns validated webhook dispatch.
  - Remote-hub owns header parsing and HTTP response mapping.
- Inputs:
  - `trigger_id`
  - `Idempotency-Key`
  - shared-secret header
  - opaque JSON request body
- Outputs:
  - A `TriggerDeliveryReport` or structured rejection.
- State Transitions:
  - `TriggerDelivery` state transitions remain unchanged.
- Error Handling:
  - Missing/invalid secret and missing idempotency key return explicit HTTP errors.
  - Duplicate requests return the existing deduped projection.
- Tech Stack Decision:
  - Reuse current `axum` ingress and runtime APIs.
- Visual Framework Impact:
  - None.
- Human Approval Points:
  - None.
- Reused Components:
  - Shared trigger substrate, delivery projection, approval, retry, and reopen behavior.
- New Abstractions:
  - Minimal webhook secret-hash validation helper.
- Trade-offs:
  - One fixed ingress shape is accepted now to keep GA webhook semantics narrow and auditable.
- Test Strategy:
  - Add failing remote-hub/runtime tests first for accepted, rejected, and duplicate webhook flows.
- ADR Needed:
  - No.
