## Delivery Note

- What Changed:
  - Added MCP-event trigger selector config and the governed `dispatch_mcp_event(...)` runtime ingress wrapper.
  - Bound event ingress to registered `McpServer` records and preserved the low-trust artifact / knowledge gate on the resulting run path.
- Why:
  - GA includes MCP-triggered automations, but the repository still needed to prove the governed event-ingress semantics before any real credentialed transport work starts.
- User / System Impact:
  - Registered MCP servers can now raise deduped events into the same governed automation delivery path, while unknown servers or selector mismatches are rejected early.
- Risks:
  - This slice intentionally stops at the runtime boundary, so it can be misread as “MCP events are solved” unless the later real-transport gap stays explicit.
- Rollback Notes:
  - Revert the MCP-event trigger schema/runtime changes together. Partial rollback is unsafe once MCP-event triggers are stored or referenced by automation records.
- Follow-ups:
