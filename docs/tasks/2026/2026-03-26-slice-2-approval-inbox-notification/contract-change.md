## Contract Change

- Change Type:
  - Cross-language Contract
- New / Updated Schemas:
  - Update `schemas/runtime/task.schema.json`
  - Update `schemas/runtime/run.schema.json`
  - Update `schemas/governance/approval-request.schema.json`
  - Update `schemas/governance/capability-descriptor.schema.json`
  - Update `schemas/governance/capability-binding.schema.json`
  - Update `schemas/governance/capability-grant.schema.json`
  - Update `schemas/governance/budget-policy.schema.json`
  - Update `schemas/observe/inbox-item.schema.json`
  - Update `schemas/observe/notification.schema.json`
  - Add `schemas/observe/policy-decision-log.schema.json`
- New / Updated Commands:
  - None. Slice 2 still exposes a Rust library API only.
- New / Updated Queries:
  - None as shared contracts. Rust runtime query APIs remain local implementation concerns.
- New / Updated Events:
  - None as formal shared event contracts in this slice.
- New / Updated DTOs:
  - `Task` grows the minimum governance fields for capability and budget evaluation.
  - `Run` grows the minimum governance references required to represent approval-driven lifecycle.
  - `ApprovalRequest`, `InboxItem`, `Notification`, and `PolicyDecisionLog` are refined from placeholders into concrete Slice 2 contracts.
- Compatibility Impact:
  - There are still no tracked external consumers, so this is a narrowing/refinement of placeholder contracts rather than a breaking published API change.
  - Slice 1 runtime tests and future consumers will need to use the refined fields and stricter object shapes.
- Affected Consumers:
  - Rust consumers under `crates/`
  - Future TypeScript consumers under `packages/`
  - Runtime contract tests in `crates/runtime/tests`
- Migration Notes:
  - Add SQLite migration `0002` for governance config, approval requests, inbox items, notifications, and policy-decision logs.
  - Existing Slice 1 databases should remain readable and migrate forward in place.
- Generation Impact:
  - None currently. The repo still does not generate Rust or TypeScript contract consumers from `schemas/`.
- Open Questions:
  - None for this slice; approval remains limited to `execution`, and acting subject remains Workspace / Project scoped.
