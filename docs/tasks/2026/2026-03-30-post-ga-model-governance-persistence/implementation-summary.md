- Structure Decision:
  - Keep provider/catalog/profile/policy persistence in `crates/governance`, and keep run-scoped `ModelSelectionDecision` persistence in `crates/runtime`.
- Why This Structure:
  - It follows the boundary frozen by the design-only predecessor package and avoids pulling transport or UI concerns into the first consumer slice.
- Reused Patterns:
  - Reuse the existing SQLite migration pattern, governance-store upsert/fetch style, runtime run-report loading, and targeted integration-test style already present in the repository.
- New Dependencies:
  - `serde_json` as a crate dependency in `crates/governance` for JSON-backed array field persistence.
  - `tempfile` and `tokio` as crate-local dev-dependencies in `crates/governance` for migration-backed integration testing.
- Error Handling Strategy:
  - Missing runs reject decision recording; duplicate per-run decision creation resolves to the already persisted record instead of silently widening behavior.
- Deferred Items:
  - Read-only transport surfaces and route assembly.
  - Shared client accessors.
  - Desktop/web model-center pages.
  - Provider connectivity and built-in tool modeling.
- Non-goals Preserved:
  - No new transport contracts.
  - No new remote-hub endpoints.
  - No capability-runtime redesign.
