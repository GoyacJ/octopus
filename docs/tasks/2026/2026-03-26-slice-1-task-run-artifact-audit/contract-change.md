## Contract Change

- Change Type: Cross-language Contract
- New / Updated Schemas:
  - Updated `schemas/context/workspace.schema.json`
  - Updated `schemas/context/project.schema.json`
  - Updated `schemas/runtime/task.schema.json`
  - Updated `schemas/runtime/run.schema.json`
  - Updated `schemas/observe/artifact.schema.json`
  - Updated `schemas/observe/audit-record.schema.json`
  - Updated `schemas/observe/trace-record.schema.json`
- New / Updated Commands: None.
- New / Updated Queries: None.
- New / Updated Events: None.
- New / Updated DTOs:
  - `Workspace` now fixes the minimum Slice 1 identity and display fields.
  - `Project` now fixes the minimum Workspace-scoped business-context fields.
  - `Task` now fixes the Slice 1 intake payload, including deterministic execution-action shape and idempotency key.
  - `Run` now fixes the Slice 1 orchestration fields for run type, attempt counters, checkpoint metadata, resume token, and timestamps.
  - `Artifact`, `AuditRecord`, and `TraceRecord` now fix the minimum persisted observation shapes used by the Slice 1 closed loop.
- Compatibility Impact:
  - No existing runtime consumers are broken because there were no tracked consumers before this slice.
  - This task intentionally narrows previously permissive placeholders into concrete Slice 1 contracts.
- Affected Consumers:
  - Future Rust consumers in `crates/`
  - Future TypeScript consumers in `packages/`
  - Slice 1 runtime tests in `crates/runtime/tests`
- Migration Notes:
  - No schema-version migration path is required yet because no prior consumers existed.
  - SQLite table creation for the new runtime slice is handled by the Slice 1 migration in `crates/runtime/migrations/`.
- Generation Impact:
  - None in this slice. No code generation or derived package publishing is introduced.
- Open Questions:
  - Approval, automation, and shared-knowledge contracts remain intentionally deferred to later slices.
